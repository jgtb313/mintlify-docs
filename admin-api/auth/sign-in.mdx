---
title: "Sign In"
api: "POST /auth/sign-in"
description: "Complete admin sign-in after OTP validation."
authMethod: "none"
---

Completes the admin passwordless sign-in after the OTP has been validated.

## Request Body

<ParamField body="otpId" type="string" required>
  The OTP request identifier.
</ParamField>

<ParamField body="code" type="string" required>
  The validated OTP code.
</ParamField>

## Response

<ResponseField name="accessToken" type="string">
  JWT access token.
</ResponseField>

<ResponseField name="refreshToken" type="string">
  Refresh token.
</ResponseField>

<ResponseField name="expiresIn" type="integer">
  Access token expiration in seconds.
</ResponseField>

## Error Codes

| Status | Code | Description |
|--------|------|-------------|
| `400` | `validation_error` | Invalid input or missing required fields |
| `401` | `auth.invalid_credentials` | Email or password is incorrect |
| `500` | `internal_server_error` | An unexpected error occurred |

## Special 403 Responses

The sign-in flow may return `403` with specific sub-codes requiring additional action:

| Code | Description | Action Required |
|------|-------------|-----------------|
| `auth.organization_selection_required` | User belongs to multiple organizations | Select an organization |
| `auth.otp_verification_required` | Email verification pending | Complete OTP verification |
| `auth.method_not_allowed` | Authentication method not allowed by organization | Use the correct auth flow |
