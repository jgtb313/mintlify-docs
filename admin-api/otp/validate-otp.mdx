---
title: "Validate OTP"
api: "POST /otps/{otpId}"
description: "Validate a one-time password code."
authMethod: "none"
---

Validates the OTP code sent to the admin.

## Path Parameters

<ParamField path="otpId" type="string" required>
  The OTP request identifier returned by [Send OTP](/admin-api/otp/send-otp).
</ParamField>

## Request Body

<ParamField body="recipient" type="string" required>
  The admin's email address or phone number. Must match the one used in Send OTP.
</ParamField>

<ParamField body="code" type="string" required>
  The OTP code received by the recipient.
</ParamField>

## Response

<ResponseExample>
```json 204
// No content — OTP validated successfully
```
```json 400
{
  "statusCode": 400,
  "message": "auth.invalid_code",
  "errors": [{"message": "required", "path": "field"}]
}
```
```json 500
{
  "statusCode": 500,
  "message": "Internal Server Error"
}
```
</ResponseExample>

## Error Codes

| Status | Code | Description |
|--------|------|-------------|
| `400` | `validation_error` | Invalid input or missing required fields |
| `400` | `auth.invalid_code` | OTP code is invalid or expired |
| `500` | `internal_server_error` | An unexpected error occurred |

## Special 403 Responses

The OTP validation may return `403` with specific sub-codes requiring additional action:

| Code | Description | Action Required |
|------|-------------|-----------------|
| `auth.organization_selection_required` | User belongs to multiple organizations | Select an organization |
| `auth.method_not_allowed` | Authentication method not allowed by organization | Use the correct auth flow |
