Authenticates a user with email and password.

## Request Body

<ParamField body="email" type="string" required>
  User email address.
</ParamField>

<ParamField body="password" type="string" required>
  User password.
</ParamField>

## Response

<ResponseField name="accessToken" type="string">
  JWT access token for authenticating API requests.
</ResponseField>

<ResponseField name="refreshToken" type="string">
  Token for obtaining new access tokens without re-authentication.
</ResponseField>

<ResponseField name="expiresIn" type="integer">
  Access token expiration time in seconds.
</ResponseField>

## Error Codes

| Status | Code | Description |
|--------|------|-------------|
| `400` | `validation_error` | Invalid input or missing required fields |
| `401` | `auth.invalid_credentials` | Email or password is incorrect |
| `500` | `internal_server_error` | An unexpected error occurred |

## Special 403 Responses

The sign-in flow may return `403` with specific sub-codes requiring additional action:

| Code | Description | Action Required |
|------|-------------|-----------------|
| `auth.organization_selection_required` | User belongs to multiple organizations | Call [Switch Organization](/studio-api/profile/switch-organization) |
| `auth.otp_verification_required` | Email verification pending | Complete [OTP verification](/studio-api/otp/verify-otp) |
| `auth.method_not_allowed` | User registered via a different auth method | Use the correct auth flow (e.g., OAuth instead of password) |
