The Studio API is the primary interface for end-users of the AWSales platform.

## Base URL

```
https://studio.api.awsales.io
```

## Authentication

The Studio API uses **JWT (JSON Web Tokens)** for authentication. After signing in, the client receives an `accessToken` and a `refreshToken`.

### Bearer Token

Include the access token in the `Authorization` header for all authenticated requests:

```
Authorization: Bearer <accessToken>
```

### Token Lifecycle

| Token | Purpose | Expiration |
|-------|---------|------------|
| `accessToken` | Authenticates API requests | Short-lived (returned in `expiresIn` seconds) |
| `refreshToken` | Obtains new access tokens without re-authentication | Long-lived |

## Key Concepts

- **Organizations** are the tenant unit. Users belong to organizations via memberships.
- **Subscriptions** link organizations to plans with billing intervals.
- **Roles & Permissions** control access using the `resource:action` pattern.
