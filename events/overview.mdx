AWSales uses **message queues** for asynchronous inter-service communication. All messages follow the **CloudEvents v1.0** specification.

## CloudEvents Envelope

Every message published to a queue uses the following envelope:

```json
{
  "specversion": "1.0",
  "type": "domain.event-name",
  "id": "evt_0195f3a2-8c00-7d4e-b800-000000000000",
  "time": "2026-03-25T14:00:00Z",
  "datacontenttype": "application/json",
  "data": {
    // event-specific payload
  }
}
```

## Envelope Fields

| Field | Type | Description |
|-------|------|-------------|
| `specversion` | string | Always `"1.0"` |
| `type` | string | Event type in `domain.event-name` format |
| `id` | string | Unique event ID (format: `evt_<uuid>`) |
| `time` | string | ISO 8601 timestamp of when the event occurred |
| `datacontenttype` | string | Always `"application/json"` |
| `data` | object | Event-specific payload |
