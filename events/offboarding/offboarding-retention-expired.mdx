---
title: "offboarding.retention_expired"
description: "Published when an offboarding request's retention deadline has passed without resolution."
---

## Event Details

| Property | Value |
|----------|-------|
| **Type** | `offboarding.retention_expired` |
| **Schedule** | Once per day |

## Payload Schema

<ResponseField name="offboardingRequestId" type="string (UUID v7)" required>
  The offboarding request unique identifier.
</ResponseField>

<ResponseField name="organizationId" type="string (UUID v7)" required>
  The organization UUID.
</ResponseField>

<ResponseField name="action" type="string" required>
  The offboarding action. One of: `CANCEL_NOW`, `START_RETENTION`.
</ResponseField>

<ResponseField name="reason" type="string" required>
  The primary reason for offboarding.
</ResponseField>

<ResponseField name="reasonDetail" type="string|null" required>
  Additional detail about the reason.
</ResponseField>

<ResponseField name="notes" type="string|null" required>
  Free-text notes.
</ResponseField>

<ResponseField name="campaignMode" type="string|null" required>
  The campaign mode applied during retention. One of: `SUSPENDED`, `INACTIVE`.
</ResponseField>

<ResponseField name="billingMode" type="string|null" required>
  The billing mode applied during retention. One of: `CHARGED`, `FREE`.
</ResponseField>

<ResponseField name="billingBehavior" type="string|null" required>
  The billing behavior applied when billing mode is `FREE`.
</ResponseField>

<ResponseField name="retentionDeadline" type="string|null" required>
  The retention deadline (ISO 8601).
</ResponseField>

<ResponseField name="retentionResolution" type="string|null" required>
  The resolution of the retention process. One of: `RETAINED`, `CHURNED`, `AUTO_EXPIRED`.
</ResponseField>

<ResponseField name="status" type="string" required>
  The offboarding request status. One of: `IN_RETENTION`, `PENDING_CANCELLATION`, `RETAINED`, `CANCELLED`.
</ResponseField>

<ResponseField name="requestedBy" type="string (UUID v7)" required>
  The user who requested the offboarding.
</ResponseField>

<ResponseField name="requestedAt" type="string" required>
  Timestamp when the offboarding was requested (ISO 8601).
</ResponseField>

<ResponseField name="resolvedBy" type="string|null" required>
  The user who resolved the retention.
</ResponseField>

<ResponseField name="resolvedAt" type="string|null" required>
  Timestamp when the retention was resolved (ISO 8601).
</ResponseField>

<ResponseField name="cancelledBy" type="string|null" required>
  The user who finalized the cancellation.
</ResponseField>

<ResponseField name="cancelledBySystem" type="boolean|null" required>
  Whether the cancellation was triggered automatically by the system.
</ResponseField>

<ResponseField name="cancelledReason" type="string|null" required>
  The reason provided when finalizing the cancellation.
</ResponseField>

<ResponseField name="cancelledAt" type="string|null" required>
  Timestamp when the cancellation was finalized (ISO 8601).
</ResponseField>

<ResponseField name="cancellationDetails" type="object|null" required>
  Additional cancellation details.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  Creation timestamp (ISO 8601).
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  Last update timestamp (ISO 8601).
</ResponseField>
